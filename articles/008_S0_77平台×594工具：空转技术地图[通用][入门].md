# 77 平台×594 工具：空转技术地图

空间转录组有 10+ 个商业平台、数十个开源平台、数百个分析工具。面对这个复杂的技术生态，如何快速定位"我的任务需要什么工具"？

本文建立"任务→工具→平台"的映射关系，给出选型决策框架。

---

## 第一部分：为什么需要技术地图

### 问题 1：平台选择困境

假设你要分析肿瘤微环境的免疫浸润模式。你面临的选择：
- Visium：55μm spot，覆盖全组织，但分辨率低
- Visium HD：2-8μm bin，接近单细胞，但数据量大
- Xenium：单细胞分辨率，但只能检测预设的 300-500 个基因
- CosMx：单细胞 + 1000 基因，但视野有限

**没有"最好"的平台，只有"最适合"的平台**。选择取决于：
- 研究问题：需要全转录组还是靶向基因？
- 分辨率需求：spot 级别够用还是必须单细胞？
- 样本类型：新鲜冷冻还是 FFPE（Formalin-Fixed Paraffin-Embedded，福尔马林固定石蜡包埋）？
- 预算与时间：成本和分析复杂度能否接受？

### 问题 2：工具选择迷雾

即使确定了平台，分析工具的选择同样复杂。

以空间域识别为例，可选工具包括：
- BayesSpace：贝叶斯模型，适合 Visium
- SpaGCN：图卷积网络，适合各种分辨率
- BANKSY：多尺度邻域，适合高分辨率
- Leiden clustering：经典聚类，简单快速

每个工具都有自己的假设、适用场景和局限性。选错工具，结果可能完全不可信。

### 问题 3：任务分解困难

"分析肿瘤微环境"这个任务太宽泛。需要分解为：
1. 质量控制：过滤低质量 spot
2. 归一化：消除技术变异
3. 空间域识别：找到肿瘤中心、侵袭前沿、免疫浸润区
4. 细胞类型反卷积：推断每个 spot 的细胞组成
5. 细胞通讯分析：识别免疫-肿瘤细胞通讯
6. 空间可变基因：发现区域特异性标志物

每个子任务对应不同的工具和方法。**技术地图的作用是建立"任务→工具→平台"的清晰映射**。

## 第二部分：平台分类与特性

### 2.1 按技术原理分类

空间转录组平台可以分为三大类：

**测序型（Sequencing-based）**
- 原理：空间条码捕获 RNA，测序后定位
- 代表：Visium、Visium HD、Slide-seq、Stereo-seq
- 优势：全转录组检测（数万基因）
- 劣势：分辨率受限于 spot/bin 大小

**成像型（Imaging-based）**
- 原理：原位杂交或测序，直接在组织上成像
- 代表：Xenium、CosMx、MERFISH、seqFISH+
- 优势：单细胞或亚细胞分辨率
- 劣势：只能检测预设基因（几百到几千）

**ROI 型（Region of Interest）**
- 原理：选择感兴趣区域，提取 RNA 后测序
- 代表：GeoMx DSP、Curio Seeker
- 优势：灵活选择区域，全转录组
- 劣势：需要预先定义 ROI，空间连续性差

### 2.2 主流平台对比表

| Platform | Type | Resolution | Genes | Tissue Size | Sample Type | Cost |
|----------|------|------------|-------|-------------|-------------|------|
| **Visium** | Sequencing | 55μm spot | ~18,000 | 6.5×6.5 mm | FF/FFPE | $$ |
| **Visium HD** | Sequencing | 2μm bin | ~18,000 | 6.5×6.5 mm | FF | $$$ |
| **Xenium** | Imaging | Single-cell | 300-500 | 10×15 mm | FF/FFPE | $$$ |
| **CosMx** | Imaging | Single-cell | ~1,000 | 600 FOV | FF/FFPE | $$$$ |
| **MERFISH** | Imaging | Single-cell | 500-10,000 | Customizable | FF | $$$$ |
| **Stereo-seq** | Sequencing | 500nm bin | ~18,000 | cm² scale | FF | $$ |
| **GeoMx DSP** | ROI | ROI-level | ~18,000 | 600 ROI | FF/FFPE | $$$ |
| **Slide-seq** | Sequencing | 10μm bead | ~18,000 | Customizable | FF | $ |

**缩写说明**：
- FF: Fresh Frozen（新鲜冷冻）
- FFPE: Formalin-Fixed Paraffin-Embedded（福尔马林固定石蜡包埋）
- FOV: Field of View（视野）
- ROI: Region of Interest（感兴趣区域）
- UMI: Unique Molecular Identifier（唯一分子标识符）
- NMF: Non-negative Matrix Factorization（非负矩阵分解）
- WLS: Weighted Least Squares（加权最小二乘法）
- NB: Negative Binomial（负二项分布）
- DSP: Digital Spatial Profiler（数字空间分析仪）
- BASS: Bayesian Analysis of Spatial Structure（空间结构贝叶斯分析）
- CNN: Convolutional Neural Network（卷积神经网络）
- COMMOT: COMMunication analysis by Optimal Transport（基于最优传输的通讯分析）
- LIANA: LIgand-receptor ANalysis frAmework（配体-受体分析框架）

### 2.3 平台选型决策树

```
Decision Tree: What is your research question?
│
├─ Need whole transcriptome?
│  ├─ Yes → Need single-cell resolution?
│  │  ├─ Yes → Visium HD (2-8μm bin)
│  │  └─ No → Visium (55μm spot) or Stereo-seq
│  │
│  └─ No → Targeted gene panel is enough
│     └─ Need single-cell resolution?
│        ├─ Yes → Xenium (300-500 genes) or CosMx (1000 genes)
│        └─ No → Visium with targeted panel
│
├─ Sample type?
│  ├─ FFPE only → Visium, Xenium, CosMx, GeoMx
│  └─ Fresh Frozen → All platforms available
│
├─ Budget constraint?
│  ├─ Limited → Visium, Slide-seq
│  └─ Flexible → Visium HD, Xenium, CosMx
│
└─ Analysis complexity tolerance?
   ├─ Simple → Visium (mature ecosystem)
   └─ Complex → Visium HD, Xenium (emerging tools)
```

## 第三部分：任务-工具映射表

### 3.1 核心分析任务分类

空间转录组分析可以分为 8 大任务类别：

1. **数据读取与预处理**：加载数据、格式转换
2. **质量控制**：过滤低质量 spot/细胞
3. **归一化与批次校正**：消除技术变异
4. **空间域识别**：发现功能分区
5. **空间可变基因**：识别空间模式基因
6. **细胞类型推断**：反卷积或注释
7. **细胞通讯分析**：配体-受体相互作用
8. **可视化与报告**：空间图、热图、统计图

### 3.2 任务-工具映射表

#### Task 1: Data Loading & Preprocessing

| Tool | Platform Support | Language | Key Features |
|------|-----------------|----------|--------------|
| **Scanpy** | Visium, HD | Python | AnnData format, standard workflow |
| **SpatialData** | Multi-platform | Python | Unified data model, HD support |
| **Seurat** | Visium, HD | R | Comprehensive, mature ecosystem |
| **Giotto** | Multi-platform | R | Platform-agnostic framework |

**推荐组合**：
- Visium 标准流程：Space Ranger → Scanpy

- 跨平台分析：SpatialData 或 Giotto

#### Task 2: Quality Control

| Tool | Method | Best For |

**关键指标**：
- Total UMI counts
- Number of genes detected
- Mitochondrial gene percentage
- Spatial distribution patterns
- In-tissue vs out-of-tissue

#### Task 3: Normalization & Batch Correction

| Tool | Method | Assumption | Best For |
|------|--------|------------|----------|

**选择建议**：
- 单样本：Log-normalization（Scanpy）
- 多样本无批次效应：Log-normalization
- 多样本有批次效应：Harmony 或 scVI
- 空间批次效应：BASS

#### Task 4: Spatial Domain Identification

| Tool | Method | Resolution | Complexity |
|------|--------|------------|------------|
| **Leiden** | Graph clustering | All | Low |
| **BayesSpace** | Bayesian model | Spot-level | Medium |

| **Giotto** | Multiple methods | All | Medium |

**选择建议**：
- 快速探索：Leiden clustering（Scanpy）
- Visium 标准：BayesSpace
- 高分辨率（HD/Xenium）：BANKSY
- 结合组织形态：stLearn

#### Task 5: Spatially Variable Genes (SVG)

**选择建议**：
- 快速筛选：Squidpy（Moran's I）
- 精细模式：SpatialDE

- 标准流程：Cell2location（准确但慢）
- 快速分析：RCTD（快速但简化）

## 第五部分：选型决策框架

### 5.1 平台选型的四个维度

**维度 1：研究问题**
- 探索性研究（不知道看什么）→ 全转录组（Visium/HD）
- 验证性研究（已知候选基因）→ 靶向平台（Xenium/CosMx）
- 细胞通讯研究 → 单细胞分辨率（HD/Xenium/CosMx）
- 组织结构研究 → 大视野（Visium/Stereo-seq）

**维度 2：样本特性**
- 新鲜冷冻 → 所有平台可选
- FFPE → Visium/Xenium/CosMx/GeoMx
- 珍贵样本（量少）→ Visium（需求量小）
- 大组织（>1cm²）→ Stereo-seq

**维度 3：资源约束**
- 预算有限 → Visium 或 Slide-seq
- 时间紧迫 → Visium（工具成熟）
- 计算资源有限 → Visium（数据量小）
- 分析能力有限 → Visium（教程多）

**维度 4：下游分析需求**
- 需要细胞类型反卷积 → 全转录组（Visium/HD）
- 需要精确细胞定位 → 单细胞分辨率（HD/Xenium/CosMx）
- 需要亚细胞结构 → 超高分辨率（Xenium/CosMx）
- 需要 3D 重建 → 连续切片（任何平台）

### 5.2 工具选型的三个原则

**原则 1：从简单到复杂**
- 先用标准方法（Scanpy + Squidpy）建立 baseline
- 遇到问题再引入专用工具
- 避免一开始就用复杂方法

**原则 2：验证工具假设**
- 每个工具都有假设（如空间平滑性、细胞类型独立性）
- 检查数据是否满足假设
- 使用诊断图验证

**原则 3：多工具对照**
- 关键结论用多个工具验证
- 比较结果的一致性
- 报告工具间的差异

### 5.3 常见场景的推荐方案

**场景 1：肿瘤微环境分析（FFPE 样本）**
- 平台：Visium（成本效益好）
- 工具链：
  - QC: Scanpy
  - Domain: BayesSpace
  - Deconvolution: Cell2location
  - CCC: CellChat
  - Visualization: Scanpy + Squidpy

**场景 2：大脑区域识别（新鲜冷冻）**
- 平台：Visium HD（需要单细胞分辨率）
- 工具链：
  - QC: SpatialData
  - Domain: BANKSY
  - SVG: Squidpy
  - Annotation: Tangram
  - Visualization: Napari

**场景 3：发育生物学（小鼠胚胎）**
- 平台：Stereo-seq（大视野 + 高分辨率）
- 工具链：
  - QC: Scanpy
  - Domain: SpaGCN
  - SVG: SpatialDE
  - Trajectory: Monocle3
  - Visualization: Scanpy

**场景 4：免疫浸润验证（已知基因）**
- 平台：Xenium（靶向 + 单细胞）
- 工具链：
  - QC: Custom scripts
  - Clustering: Scanpy
  - CCC: Squidpy
  - Visualization: Napari

## 第六部分：证据链与边界声明

### 6.1 工具选择的证据链

选择工具不是拍脑袋，需要证据支持：

**证据 1：Benchmark 研究**
- 查阅工具比较文章（如 Nature Methods 的 benchmark）
- 关注准确性、速度、稳健性指标
- 注意 benchmark 的数据类型和场景

**证据 2：方法论文**
- 阅读工具的原始论文
- 理解方法的假设和局限性
- 检查是否适用于你的数据

**证据 3：社区反馈**
- GitHub issues 和 discussions
- 论坛（如 Biostars、单细胞中文社区）
- 文献中的使用频率

**证据 4：实际测试**
- 在自己的数据上测试多个工具
- 比较结果的生物学合理性
- 评估计算成本和易用性

### 6.2 工具选择的对照验证

选择工具后，需要通过对照验证结果的可靠性：

**对照 1：多工具对照**
- 使用 2-3 个同类工具分析相同数据
- 比较核心结论的一致性
- 示例：空间域识别用 BayesSpace + SpaGCN + Leiden，看分区是否一致

**对照 2：参数敏感性**
- 改变关键参数（如聚类数、邻域半径）
- 观察结果的稳定性
- 不稳定的结果需要谨慎解释

**对照 3：负对照**
- 使用随机化数据或打乱空间坐标
- 预期不应出现空间模式
- 如果仍有"显著"结果，说明方法有问题

**对照 4：已知标志物验证**
- 用已知的组织结构或细胞类型标志物验证
- 空间域是否与组织学结构一致？
- 反卷积结果是否与 marker 基因表达一致？

### 6.3 边界声明

**本文能说明什么**：
- 主流平台和工具的分类与特性
- 任务-工具-平台的映射关系
- 选型的决策框架

**本文不能说明什么**：
- 哪个工具"最好"（取决于具体场景）
- 工具的详细使用方法（需要专门文章）
- 最新的工具（生态快速演化）

**使用建议**：
- 本文是导航，不是教程
- 选型后需要深入学习具体工具
- 定期更新工具知识（每 6 个月）

## 总结

空间转录组的技术生态复杂但有规律：
- **平台选择**：基于研究问题、样本类型、资源约束、分析需求四个维度
- **工具选择**：基于任务分解、工具特性、假设验证、多工具对照
- **推荐策略**：从简单到复杂，先建立 baseline，再优化

**核心工具栈推荐**：
- Python：Scanpy + Squidpy + SpatialData（覆盖 80% 需求）
- R：Seurat + Giotto（传统生信用户）
- 专用工具：根据任务按需引入

技术地图不是静态的。随着新平台和工具的出现，需要持续更新知识体系。但核心的选型逻辑——"任务驱动、证据支持、多工具验证"——是不变的。
